See It In Action

01 / 04

Instant Context

Ask about something you discussed 3 weeks ago. Velophos retrieves the exact context using semantic similarity — not keyword search.

await velophos.chat("What was my last project idea?")
// Retrieves: "Python CLI tool for log analysis" — from 3 weeks ago
02 / 04

Multi-modal RAG

Feed it PDFs, URLs, or raw text. It chunks, embeds, and retrieves relevant passages automatically — your private knowledge base.

await velophos.ingest({ source: "./whitepaper.pdf" })
await velophos.query("Summarize key findings")
03 / 04

Streaming Responses

Token-by-token streaming over WebSockets or Server-Sent Events. No waiting for the full response — it feels alive.

const stream = velophos.stream("Explain quantum entanglement")
for await (const token of stream) process.stdout.write(token)
04 / 04

Fine-Tune Ready

Phase 2 includes LoRA fine-tuning on your own conversation data. Transform the base model into your domain-specific expert.

await velophos.finetune({
  dataset: "./my-conversations.jsonl",
  epochs: 3, method: "lora"
})
The System

How Velophos Thinks

01

Receive & Embed

Your message is embedded into a 384-dimension semantic vector using sentence-transformers running locally.

02

Memory Retrieval

ChromaDB performs cosine similarity search across your entire conversation history to retrieve the most relevant context.

03

Context Assembly

Retrieved memories + current message are assembled into an optimized prompt that fits the model's context window.

04

Inference & Stream

The local LLM generates a token-streamed response. The exchange is then embedded and stored back into memory.
